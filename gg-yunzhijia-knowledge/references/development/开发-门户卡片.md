this.$on('notice', function () {

方法

this.hide()

this.setSubTitle('卡片副标题')

刷新数据

重新拉取卡片数据（仅在配置了卡片数据来源时生效）

this.reloadData()

发起请求

发起ajax请求（参考卡片获取后端数据）

当配置卡片数据来源无法满足卡片取数据的要求时，可以通过该方法手动获取卡片数据。

通过该方式发起的请求无session，但是会附带一个ticket，前提是该卡片设置了appid。

this.request({
    url: 'https://mydomain.com/interface',
    method: 'post',
    serializer: 'json',
    headers: {},
    data: {
      a: '123'
    }
}).then(function (result) {
  console.log(result)
})

获取ticket

在卡片中获取ticket

this.getTicket(appId).then(function (ticket) {
	console.log(ticket)
}).catch(function () {
	console.log('获取ticket失败')
})

## 简介

卡片简介

什么是门户卡片

门户由多张卡片组成，每张卡片是一个独立的前端组件，多张卡片可以实现APP和多个业务系统之间的连接。

与普通Vue组件一样，可以用template和style完全自定义卡片的样式，用mixins定义卡片的属性、方法和事件，上手简单。了解Vue语法

JSSDK

在卡片中可以使用jssdk访问客户端API。

在卡片中不需要引用qing.js，直接使用window.qing变量即可，部分桥不需config，若调用失败执行qing.config()后再试。

// 仅web端
qing.config({
    jsApiList: ['aaa', 'bbb']
})

在线开发

卡片开发者 无需安装开发环境，直接在门户管理后台就可以完成卡片的开发和上线。

卡片获取后端数据

卡片只能通过https接口与后端系统进行通信，该接口由业务方自行开发，需返回约定格式的json结果，例如：

{
    "success": true,
    "data": {
        "list": [
            {
                "title": "4企业IPO申请上会 仅1家过会",
                "day": "2017-12-21",
            }, {
                "title": "嘉实基金李化松：环保低碳行业投资空间巨大长期有望获取超额收益",
                "day": "2017-12-21",
            }
        ]
    }
}

其中，success 和 data 是必须的， 而 data 里面的内容是可以自由定义的。

注意，因为浏览器对AJAX请求有跨域限制，所以卡片如果要在Web端展示，需要给卡片获取数据的接口加上允许跨域的响应头：

Access-Control-Allow-Origin: *

卡片后端接口获取用户身份

如果卡片配置了appId，该卡片的请求数据接口中会自动带上客户端生成的 ticket，服务端可以通过解析ticket解析出用户身份。

引入npm模块

借助于 unpkg.com 提供的在线模块，可以在卡片的 mounted() 事件中使用 tools.use(name) 方法来引入开源组件。

也可以使用 tools.loadScript(src) 方法来引入外部 js，但要注意门户页面有CSP同源策略，公有云只能引入www.yunzhijia.com域名下的 js 资源，私有云则只能引入该私有云域名下的js资源。

例如：

tools.use('jquery').then(function () {
    // 现在可以使用window.jQuery了
});

## 组件

卡片组件

卡片内组件

在卡片内，开发者可通过html和css语法实现任意的样式效果，并可以通过qing.js提供的桥实现轻应用的唤起和页面跳转以及数据请求。

但如果要在卡片上实现更复杂的效果则还需要付出一些时间，为了节省开发者的时间，我们封装了几个较为常用的的组件，这些组件直接在<template>标签内引用即可。

横向滑动组件 Swiper

实现左右滑动切换卡片的效果

移动端

桌面端

注意

由于目前移动端与桌面端组件部分参数不统一，需要开发者为其做相应的兼容处理。

移动端

当 totalPage > paginationMax 时，底部指示器会从点状 变成 横条

示例

<swiper :totalPage="data.length" :loadedPage="1" :paginationMax="5">
    <div v-for="(item, key) in data" :key="key">
      {{item.title}}
    </div>
  </swiper>

请求和跳转组件 Action

可以触发卡片配置界面的 action 列表中的跳转或请求动作。（参考：卡片获取后端数据）

注意

name 需要与卡片配置中字段 actions 的选项相对应

params 是可选项，可以动态绑定模板数据，作为跳转的参数

示例

<action name="showMore" :params="'id=' + item.id">
    打开应用
  </action>

页面嵌入组件 Container

该组件是对iframe的封装，如果need-ticket参数设置为true，则会在src后自动带上ticket参数

示例

<container :need-ticket="true" src="https://www.yunzhijia.com"></container>

卡片刷新组件 Sync

该组件用于触发卡片重新请求数据源接口

用法:

<sync mode="autoSync" :interval="60"></sync>

注意

参数 interval：单位为秒，只在 mode="autoSync" 时有效。

<sync>标签放在任意位置均可生效，但需注意template中只能有一个根节点

使用示例：

<div class="my-card">
  <sync mode="autoSync" :interval="60"></sync>
  <div>other code</div>
</div>

## 门户卡片开发

门户卡片开发

卡片开发概述

卡片是门户的主要组成部分，你可以把一个卡片看做一个简化版的轻应用。也可以吧一个门户看做是多个轻应用组成的控制台。

从技术角度来说，每张卡片都是一个独立的前端组件，多张卡片可以实现多个轻应用和多个业务系统之间的连接。

卡片介绍

使用Vue语法

与普通Vue组件一样，可以用template和style完全自定义卡片的样式，用mixins定义卡片的属性、方法和事件，上手简单。了解Vue语法

jssdk

在卡片中可以使用jssdk访问客户端API，点击此处 可以查看使用方法。

注意：卡片不是独立页面，因此在卡片中不需要引用qing.js，也不需要执行qing.config()，直接使用window.qing变量即可。

在线开发

卡片开发者无需安装开发环境，直接在门户管理后台就可以完成卡片的开发和上线。如下流程：

卡片开发流程

1、工作圈管理员或者门户管理员创建卡片

在门户管理后台，选择【门户卡片开发管理】，然后再选择【团队卡片开发】：

点击右上角【+ 新建】按钮快速创建卡片，填写相应卡片名称与详细描述，然后选择卡片相应的开发者，通知开发者登录并开发卡片：

2、开发者编辑卡片

当管理员分配开发者后，开发者登录管理后台，然后在【团队卡片开发】页面找到该卡片点击【设置】按钮开始编写卡片的配置。

智能门户的卡片开发和用户在卡片开发平台(https://www.yunzhijia.com/card-platform/login)上开发的卡片，方法和步骤总体来说是差不多的，智能门户的卡片是在原卡片开发平台的基础上增加了设置项的配置，该功能适合复杂多变的卡片。

原卡片(卡片开发平台的卡片)可以直接粘贴至新门户，可能只需要您稍微调整一下卡片样式。

基础配置

1、卡片图标

为卡片上传一个 logo 图标，推荐使用 svg 格式，此图标会出现在卡片运行时的左上角

2、轻应用 ID

这个轻应用 id 就是卡片所对应的轻应用的 appId，可以在应用管理后台查阅各个应用的 appId

3、是否需要鉴权

*仅高代码卡片支持此功能*

勾选此项后，在卡片内部发起 ajax 请求时，工作台会在请求的 url 后面带上 (?|&)ticket=${ticket} 参数， 以便通过后端服务身份校验。注意 ticket 的获取必须要有 appId，所以只有填写了上述的 appId 后，勾选此项才会生效

4、卡片支持的端

可选择卡片仅在部分端可添加进布局。对支持的端再次修改时，历史已存在运行时的卡片将不会改变，需要门户管理员手动删除历史卡片，并重新添加卡片到布局，才会显示最新的配置效果。

5、业务名称

*仅配置型卡片支持此功能*

若配置型-入口类卡片支持门户管理员在门户编辑页选择业务数据源，此名称标识卡片所属业务名称。下方红色框所示即是配置后的效果：

6、卡片右上角按钮

*仅高代码卡片支持此功能*

可以选择是否显示卡片的右上角按钮，用户点击卡片右上角按钮一般会跳转到轻应用主页， 所以需要在下面填写按钮的文本，并选择下述接口配置——接口列表中填写好的跳转url。

7、卡片无数据显示规则

*仅高代码卡片支持此功能*

用来设置当卡片没有数据时, 卡片该如何在三端展示。卡片是否有数据, 需要卡片开发者自己判断, 三个配置项 显示统计内容、

显示缺省内容、自动隐藏都需要卡片开发者自己在卡片组件中实现，其中自动隐藏可以通过 this.$emit('hide') 实现。

接口配置

*仅高代码卡片支持此功能*

1、接口列表

可以点击添加Action按钮增加一个接口或者跳转url，接口或者url按照表格的提示填写好相应的内容。这里的接口主要是让后面的卡片初次加载数据接口和清除卡片红点接口这两项来使用的，而跳转url则是让卡片右上角按钮来使用的。

2、卡片初次加载数据接口

这里可以选择前面在接口列表中填写的接口, 此接口会在两种情况下调用

卡片初始渲染前

工作台收到卡片业务后台的新数据推送时：调用后, 接口的返回数据会通过 props 传递给卡片组件, 卡片卡发者可以在卡片组件的 props 中声明 data 属性, 从而拿到接口的返回数据

3、清除卡片红点接口

当卡片业务后台向工作台推送新数据时，可以通过一个字段告知工作台是否在卡片上显示红点。 显示红点表示此卡片有新数据到来，工作台会把有红点的卡片优先靠前展示。当用户浏览过有卡片的红点后, 需要清除红点， 这时就需要指定一个后端接口，工作台会调用此接口来清除红点。这个接口就需要在前面的接口列表里填写好。

tab配置

*仅配置型卡片支持此功能*

【简介】当页面空间有限时，卡片需要展示多维度且有关联性的信息时，可以选用tab组件来展示；

【场景示例】A企业应用多，希望能对应用进行分类且在一张卡片上完整显示所有分类，应用汇总方便用户查找；

【解决方案】卡片开发者可设置支持多tab模式，门户管理员场景卡片时可自定义tab数量与tab下的数据源；

【方案示意图】

1、支持tab设置

依据卡片自身业务需要开启此开关。若不开启，此卡片在设计时将不支持门户管理员设置tab；开启后，有两种方式可设置tab，见下述

2、tab设置方式

2.1、设计时门户管理员自定义

门户管理员可根据业务实际需要，在门户编辑页，设置当前卡片是否支持设置tab。

2.2、通过接口获取tab数据

【简介】业务方自身已有tab内容和tab下的数据源，无需在门户设计时定义tab内容；

【场景示例】A企业希望应用中心的数据展示在门户卡片上，同时希望以应用中心的分类进行展示；

【解决方案】卡片开发者可设置通过接口获取tab数据，门户管理员只需添加该卡片即可展示接口的数据；

【方案示意图】

开发者按照如下格式返回接口数据：

内容配置

1、内容数据来源

*仅配置型卡片支持此功能*

智能门户可展示各个业务的数据，业务方可开发卡片，可设置卡片可选择的数据源

1.1、设计时门户管理员自定义

此项设置产生的影响同上述tab配置中的设计时门户管理员自定义，此处不再赘述。

若当前卡片为配置型--入口类卡片，还可进一步选择在门户编辑页中，卡片内容可选择的不同入口类型，包括以下三种：

业务数据源：门户管理员可选择业务数据源作为入口

入口可指定url：门户管理员可自定义入口的URL

可设置入口不跳转：门户管理员可设置入口不跳转，仅做展示

上述三种入口类型分别对应下述入口内容设置中跳转至的三种类型：

1.2、通过接口获取tab数据

此功能类似上述tab配置中的通过接口获取tab数据。不同的是，此处不需要通过接口返回配置数据，在数据展示配置（下述）中即可交互式完成配置。因此可知，此种情况下卡片内容是无tab的。

2、数据源选择器

*仅配置型卡片支持此功能*

【简介】不同卡片支持绑定不同的数据源；

【场景示例】A企业有多种类型的资讯，如：公司要闻专栏、通知公告专栏、一线动态专栏、党建专栏等，希望能创建多张卡片，不同卡片展示不同专栏的内容；

【解决方案】知识中心卡片支持场景多张卡片，不同卡片绑定不同专栏的数据源展示对应专栏的数据源，卡片绑定的数据源选择器，按照知识中心目录树的方式展示，每个层级的目录都可选择，支持不同卡片绑定不同目录；

是否为叶子节点

若左侧数据为树形数据，开发者需要明确告知当前节点是否为叶子节点。此处需自定义开发。

function (n) {

支持批量添加

*仅配置型-入口类卡片支持此功能*

开启此功能后，将可以一次性选择多个业务数据源作为入口，名称及图片均使用数据源自带的，此时不可批量修改名称及入口图片，但可在添加成功后逐一按需修改。

卡片样式库

1、高代码类型卡片样式库

卡片样式库分为：网页/桌面端和移动端两套设置, 每套设置分为默认样式和样式一/二/三。

在网页/桌面端的样式设置里，需要设置网页/桌面端的卡片推荐大小，以便管理员在卡片布局时，拖动卡片至合适的大小尺寸。 一张卡片可能有多个合适的尺寸，所以推荐尺寸可以任意添加。每种样式还需要上传一张效果预览图，并且除了默认样式以外， 其他样式都需要指定样式名称。

效果预览图和样式名称会出现在门户内容设置的卡片管理中，方便管理员选择他想要的卡片显示样式。在卡片运行时， 在卡片组件中可以通过this.card.skin.mobileSkinsValue.key 和 this.card.skin.pcWebSkinsValue.key 分别拿到移动端和网页/桌面端的选定的样式值。其中，默认样式的值为 'default', 其他样式的值为 'skin1'、'skin2'、 'skin3'，根据设置好的样式值，就能在卡片组件中实现对应的样式了。

对于网页/桌面端，卡片样式库里的每种样式只是给出了卡片推荐大小，实际的尺寸需要管理员在门户内容设置中的布局器中拖动卡片来确定。

在卡片运行时，可以在卡片组件中可以通过 this.card.webLayout.width，this.card.webLayout.height和

this.card.pcLayout.width，this.card.pcLayout.height 分别获取网页端和桌面端的卡片实际尺寸 (值为整数)。

2、配置型卡片样式库

卡片样式库分为：网页/桌面端和移动端两套设置。

配置型卡片样式库由系统内置，无法自行新增。有免费样式及收费样式（需开通门户高级特性）之分。

系统已设置一个免费的样式作为默认样式，开发者可根据自身业务需要设置其他免费样式为默认样式。

卡片代码除了分为网页/桌面端和移动端两种运行时类别外，还有一个演示数据代码需要编写。编写这部分代码的目的，是为了在卡片设计时的布局器中，能够实时显示卡片的内容和样式 (只不过数据是假的)，方便管理员后期识别卡片，拖动卡片布局，防止卡片在经过拉伸布局后发生变形，影响美观。

id: 卡片的 id

name: 卡片的名称

platform: 卡片当前所处的平台, 分为 mobile, web, pc

skinValue: 卡片当前设置的样式, 比如 default, skin1, skin2...

size: 卡片在web/pc布局器中的实时尺寸, 比如: {x: 3, y: 3}

customOptions: 卡片自定义设置项, 任意对象

# 卡片简介

> 来源：开放平台线上文档 (https://open.yunzhijia.com/opendocs/docs/card/index.md)

## 什么是门户卡片

门户由多张卡片组成，每张卡片是一个独立的`前端组件`，多张卡片可以实现APP和多个业务系统之间的连接。

与`普通Vue组件`一样，可以用`template`和`style`完全自定义卡片的样式，用`mixins`定义卡片的属性、方法和事件，上手简单。[了解Vue语法](https://cn.vuejs.org)

## JSSDK

在卡片中可以使用jssdk访问客户端API。

在卡片中不需要引用qing.js，直接使用window.qing变量即可，部分桥不需config，若调用失败执行qing.config()后再试。

```
// 仅web端
qing.config({
    jsApiList: ['aaa', 'bbb']
})
```

## 在线开发

卡片开发者 `无需安装开发环境`，直接在门户管理后台就可以完成卡片的开发和上线。

## 卡片获取后端数据

卡片只能通过`https接口`与后端系统进行通信，该接口由业务方自行开发，需返回约定格式的`json`结果，例如：

```json
{
    "success": true,
    "data": {
        "list": [
            {
                "title": "4企业IPO申请上会 仅1家过会",
                "day": "2017-12-21",
            }, {
                "title": "嘉实基金李化松：环保低碳行业投资空间巨大长期有望获取超额收益",
                "day": "2017-12-21",
            }
        ]
    }
}
```

其中，`success` 和 `data` 是必须的， 而 `data` 里面的内容是可以自由定义的。

注意，因为浏览器对AJAX请求有跨域限制，所以卡片如果要在Web端展示，需要给卡片获取数据的接口加上允许跨域的响应头：
```
 Access-Control-Allow-Origin: *
```

## 卡片后端接口获取用户身份

如果卡片配置了appId，该卡片的请求数据接口中会自动带上客户端生成的 ticket，服务端可以通过[解析ticket](/server-api/auth/index)解析出用户身份。

## 引入npm模块

借助于 [unpkg.com](https://unpkg.com) 提供的在线模块，可以在卡片的 mounted() 事件中使用 `tools.use(name)` 方法来引入`开源组件`。

也可以使用 `tools.loadScript(src)` 方法来引入外部 js，但要注意门户页面有CSP同源策略，公有云只能引入`www.yunzhijia.com`域名下的 js 资源，私有云则只能引入该私有云域名下的js资源。

例如：
```js
tools.use('jquery').then(function () {
    // 现在可以使用window.jQuery了
});
```
