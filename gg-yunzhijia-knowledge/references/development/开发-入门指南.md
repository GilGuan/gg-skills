## 开发一个小程序

开发一个小程序

小程序开发概述

什么是小程序

小程序是运行在云之家的媲美 Native 性能的应用！

通过云之家专为小程序打造的运行环境和开放接口，仅使用前端 HTML、JS、CSS 技术，即可开发出媲美 Native 性能的应用，无论是加载速度还是使用体验，都远超以往H5，极大的提高企业业务运转效率

与传统 H5 的区别

打开速度更快

传统 H5 每次打开时均需要向网络请求程序代码等资源，小程序则仅需要加载一次

切换页面无需过多等待

离线缓存的机制，不需要重复拉取程序资源，仅需请求JSON数据

基本无白屏、卡顿

传统 H5 使用过程中往往容易遇到白屏、卡顿，这都是受限于传统 H5 的地方

更接近原生体验

特殊的机制和环境，使得小程序更像原生程序

小程序的特色

高加载速度

针对小程序而设计的机制，提供更快的加载，助力企业业务高效运转

得益于云之家小程序程序托管和离线缓存等能力，使得小程序缓存在云之家本地中，仅首次需要加载

优异的性能

专用运行环境、云之家原生能力，媲美 Native 体验

专为小程序打造的运行环境，配合云之家原生能力，使得无论是运行速度还是各方面，都接近 Native 体验

一站式开发环境

帮助企业完成开发闭环，提高团队开发效率

从代码Coding、 到应用管理和审核上架等流程，组成了云之家小程序一站式开发环境，化繁为简，助力企业高效运转

什么是轻应用

轻应用是运行在云之家客户端或Web门户中的网页程序。

平台级轻应用是云之家提供的通用应用，包括智能审批、智能考勤、时间助手、智能会议等。它们可以开箱即用，也可以与企业已有业务系统进行对接。

企业自建应用是企业针对自身业务创建的，它与平台级应用一样，也可以在云之家客户端中运行并利用云之家客户端能力和用户身份。

轻应用如何运行

轻应用开发概述

名词解释：

云之家WebView： 是一个标准浏览器组件，为网页代码提供运行环境。

云之家客户端能力： 包括客户端方法、界面和事件。这些能力只能通过JSSDK（qing.js）来访问。

云之家基础接口： 包括鉴权、获取人员组织等基础接口，以及智能审批等业务接口。

JSSDK

qing.js 是轻应用或卡片和客户端之间的桥梁。

通过它，轻应用或者卡片可以调用客户端的函数、唤起客户端的界面，或者监听客户端的状态变化。

调用客户端API

点击此处查看全部客户端方法

通过 qing.call(方法名, 参数) 可以调用客户端的接口，例如：

// 获取个人信息
qing.call('getPersonInfo', {
    success : function(res){
        alert(JSON.stringify(res))
    }
});

上面例子中的 getPersonInfo 是客户端提供的方法之一，它可以获取登录当前客户端的用户的身份信息。

每个客户端接口除了自身必须的参数外，还支持以下通用参数：

success：接口调用成功时执行的回调函数。

error：接口调用失败时执行的回调函数。

complete：接口调用完成时执行的回调函数，无论成功或失败都会执行。

以上所述的回调函数的参数说明，如 success : function (res) { ... }，这里的 res 为对象，除了每个接口自身返回的数据之外，还有两个通用属性 code 和 errMsg，其值格式如下：

成功时返回结果示例：{code: 200, errMsg: "", data: {}}

失败时返回结果示例：{code: 500, errMsg: "不合法的图片文件大小", data: {}}

监听客户端事件

通过 qing.on(事件名, 回调) 可以监听客户端的事件，例如：

qing.on('appear', function () {
    alert('回到当前页面窗口')
})

目前支持的事件：

appear: 回到当前页面窗口

disappear: 离开当前页面窗口

networkAvailable: 网络连接变为可用

networkDisAvailable: 网络连接变为不可用

titleDoubleClick: 标题栏双击

keyboardChange: 键盘状态发生变化

3.5卡片内获取`ticket`的方法

首先，我们要搞清楚ticket是什么，ticket能干什么，我们为什么要获取ticket？

<font color='#768893'>通常，在云之家环境内，如果想访问某个轻应用。直接打开轻应用首页即可，因为云之家环境有能力获取该应用对应的ticket，并拼接在轻应用首页url中；

但是对于第三方应用来说，虽然使用云之家账号体系，但是没有途径获取ticket，因此在打开轻应用首页时无法通过身份验证； 所以要面向使用云之家账号体系的应用，提供一个获取ticket的服务，走网关鉴权；</font>

简单来说如果没有ticket，卡片或者轻应用不知道使用者的身份。我们需要约定一个参数ticket来把你的身份信息告诉服务端，服务端解析ticket从而解析出用户身份

例如我们想从跳转到轻应用，直接window.open（‘轻应用地址’）。没有通过身份信息校验，轻应用肯定会拒绝访问。所以我们可以跳转地址后面带上ticket：

this.getTicket().then(ticket => {
	window.open(‘轻应用地址’ + '&ticket=' + ticket)
})

## 扫码登录第三方系统

扫码登录第三方系统

扫码登录

在页面中嵌入或跳转一个二维码页面，用户通过云之家APP扫扫码成功后跳转到目标页面并在url中附带ticket，目标页面通过解析ticket以获得用户身份信息。

以下是详细过程：

第一步：创建应用

为了实现扫码自动登录的功能，你必须创建一个专门用来扫码登录的应用。这跟创建普通应用没什么差别，只要记得把移动端访问地址设置为 https://www.yunzhijia.com/opencloud/openthird/qrlogin 就可以了。

另外，记得保存好 appSecret。

## 推送一条待办通知

推送一条待办消息

简单介绍

应用可以把需要用户处理的业务信息通过待办的方式推送到消息列表，用户点击待办消息后可以进入对应的应用中完成处理。

云之家待办系统中，单据有 2 个状态， 待处理和已处理，每条单据只会处于 2 者中的一个。

它们的数量和入口会一直显示在 消息 页签的顶部，如下图：

待办系统中的单据，每一条必须包含以下 3个属性，

openId  : 接收人

appId : 轻应用 id，比如智能审批，签到，用于说明单据的来源系统

sourceId : 源id , 可为任意值，一般为对接的业务系统中具体某一条单据的 id

待办的唯一性是通过 openId + appId + sourceId  三者联合起来作为主键来保证的

前期准备

获取 accessToken 用于鉴权，获取方式请点击此处

待办的授权级别为 <font color = "red">app</font> 级，轻应用 secret 是通过管理员账号进入 管理中心--应用管理--应用详细 处获取

待办创建和流转的 2 种模式

针对待办的创建和流转，云之家平台提供两种高可靠方案供调用方选择，一种是同步模式，一种是异步模式。模式的选择是通过 创建和 流转时的参数 sync 来控制，true -同步，false-异步

同步模式

调用方先发起请求，云之家将对应的单据创建或流转后，在 http 响应里面说明此次请求是否成功

异步模式

调用方发起请求后，立刻会收到请求响应，该响应只能说明云之家平台是否受到了用户的请求，<font color='red'>不能说明单据是否创建/流转成功</font>，用户需后续不断调用待办消息确认接口检查状态，若发现失败，则进行重试

当用户请求量较大时，可以使用异步模式。同步模式虽然性能稍差一点，但简化了流程的逻辑。

>建议使用<font color='red'>同步</font>模式，但是当一次请求生成待办数在100条及以上时，建议使用异步

什么是快捷审批

对于审批的单据，用户在处理时，需要先点击单据，跳转到相应的系统中，然后进行审批处理。然而实际工作中，会存在某些不太需要“仔细”审核的单据，比如审核请假流程等。

为了提高用户工作效率，对于支持快捷审批的单据，用户在云之家内直接点击同意按钮，即可立刻将单据处理完毕。

如何接入快捷审批

建议接入类型： 用户需要及时查看处理的，并且处理能推动业务的应用消息，如流程审批、请假审批等。

接入方法：

1.开发者登陆应用管理后台，将自己的应用勾选为【审批类】，此时会生成一个signKey值，该signKey值用于表明请求是来自于云之家，开发者在接收审批请求时，需要验证 signKey是否合法，从而保证数据安全。

2.创建待办时加入参数 passUrl ，该 url 即为快捷审批专用

3.业务需要实现 passUrl: https://{domain}/xxx/batchDealTodo 对应接口，完成批量审批处理逻辑

常见问题

1.为什么待办 创建 时请求的响应返回了  成功（success)，但是用户却没有在云之家收到待办？

首先，请检查使用的模式是否为异步。异步模式返回的success<font color="red">只表示云之家收到了请求，不表示最终处理结果</font>，建议出现问题时，先改为同步模式

其次，云之家待办使用 openId + appId + sourceId  三者联合起来作为主键，当创建时发现主键重复，会自动忽略，因此，可以先用 待办查询接口，检查下该单据是否存在，若存在，则建议更换参数，避免重复。

2.我的轻应用配置为审批类，但是我想用同一个`appId` 发送通知类的消息，该如何操作？

创建待办时有个参数 todoType，用来表明单据目的地。若在创建时指定了，则会以创建的为准，若未指定，则会读取轻应用的配置信息。因此，可以在创建时指定 todoType 为 0

> 若想使用通知类的轻应用，发送审批类的待办，也是一样的道理

3.用户点击“待审批”，查看了某条消息，为什么“待审批”上的未读数仍然没有清除？

云之家待办平台可开启强提醒模式，开启后，若待办消息未处理完毕，该未读数会一直保持，用于增强提醒功能。

开启前提：在手机系统设置开启云之家消息推送；

设置路径：（移动端）头像-设置-新消息通知-顶部强提醒模式

> 强提醒功能开启后，该用户的所有团队都会设置为相同的模式

4.待办流转的请求如果在创建之前发出了，会有什么问题？

收到某条单据的流转请求后，云之家待办平台会查询该单据是否存在，若不存在，即为这个场景，此时，待办平台会将该单据的流转状态缓存 一个小时。

若在这一小时内，收到了创建请求，则会自动将该单据直接变为 流转请求里面的结果。

若超过了一个小时，缓存到期，则系统直接将流转请求从缓存中删除，回到初始未收到该单据的状态

5.为什么我点开了某条审批单据，发现这条单据已经处理完了，但云之家上仍然存在？

主要的原因是业务系统处理完单据后，例如变完成，取消，撤回等等操作时，未及时通知云之家待办平台，即未发送待办流转的请求，请联系该业务系统负责人

> 用户可以点击该单据右上角 忽略，将这条单据先去掉，避免干扰

6.推送给了一条单据给某个用户，后面撤回了，但是最后又提交给了这个用户，该如何操作？

该问题的本质是主键重复了。待办系统的主键是  openId + appId + sourceId ，调用方一般会将单据的 id 作为sourceId，当同一个流程 多次推送给同一个用户时，可能会出现这个问题。

> 云之家待办平台的单据，状态不可逆，只支持待处理-> 已处理，不支持 已处理->待处理

解决的方法：

1） 遇到这种情况时，变更sourceId

> 云之家待办平台 不会对 sourceId 做任何校验，因此任意字符都是可以的

2） 先删除旧 待办，检查到删除成功后，重新推新的待办

> 删除旧待办建议使用同步模式，根据返回的响应判断是否删除成功

良好的数据操作能力

表单数据解析步骤较为繁琐。cloudflow-sdk提供了一种良好的表单数据访问能力，该能力提供了一个表单访问器，基于该访问器可以快速访问表单数据(主表单和明细表)。开发者在接收到数据推送后将推送密文交给Cloudflow去进行解密，并得到一个Form对象，该Form对象提供根据控件CodeId、控件Title获取控件值，还可以获取一些基本参数，比如单据实例id、推送事件类型。示例代码如下：

String cipher; // 从开发者选项接收到的推送数据原文
Configuration config = new Configuration();
configuration.setCloudflowKey("xxx");
Cloudflow cloudflow = new Cloudflow(configuration);
Form form = cloudflow.decryptAsForm(cipher);
form.getWidgetValueByTitle("姓名");
form.getWidgetValueByCodeId("Ps_0");
form.getDetailWidgetValueByCodeId("Dd_0", "Te_0");
form.getDetailWidgetValueByTitle("明细1", "单行文本");

便捷的参数校验能力

数据互联控件的返回数据结构较为复杂，只有满足指定规则的返回数据才能被数据互联控件正确解析。cloudflow-sdk提供了一个ApiIntegrationResult对象，基于该对象可以便捷的构建出符合数据互联控件要求的接口返回值。示例代码如下：

ApiIntegrationResult air = new ApiIntegrationResult();
   
    List<ApiIntegrationResult.Head> heads = new ArrayList<>();
    ApiIntegrationResult.Head head = new ApiIntegrationResult.Head();
    head.setColEnName("name");
    head.setColZhName("姓名");
    head.setShowName(ApiIntegrationResult.ShowName.YES);
    head.setWidgetType(ApiIntegrationResult.WidgetType.TEXT_WIDGET);
    heads.add(head);

    ApiIntegrationResult.Head head2 = new ApiIntegrationResult.Head();
    head2.setColEnName("id");
    heads.add(head2);
    
    air.setColList(heads);

    List<Map<String, Object>> data = new ArrayList<>();
    Map<String, Object> map = new HashMap<>();
    map.put("name", "这是我的名字");
    map.put("id", "1");
    data.add(map);
    air.setDataList(data);

    air.check(); // 校验数据合法性

针对常用接口，cloudflow-sdk进行了一系列简单的封装，开发者可以无需知晓内部接口调用细节即可快速调用智能审批开放能力。开发者所需要做的只是提供配置信息并初始化一个Cloudflow对象。示例代码如下：

轻应用通过读取 navigator.userAgent 中的 fontScale 参数可以取得一个数值，把页面字体大小乘以这个值，就可以实现页面字体大小跟随APP设置变化而变化了。

// 从 UserAgent 中读取 fontScale 并转为整型
var fontScale = parseInt(getUserAgentValue('fontScale')) || 1;

// 假设你的页面原来的基准字体大小是15
var fontSize = 15 * fontScale + 'px';

document.body.style.fontSize = fontSize;

## 通过cloudhub协议url唤起第三方APP

唤起第三方APP

通过cloudhub协议url唤起第三方APP

从 10.3.2 版本开始，云之家应用中心和常用应用支持通过配置跳转协议打开第三方APP，并支持单点登录。具体方式如下：

1.配置轻应用信息

企业管理员在云之家管理中心-轻应用管理中新建轻应用（截图），同时配置轻应用的跳转协议（截图），跳转协议格式为：

>cloudhub://nativeapp?scheme=xxx

其中，scheme填写第三方APP的跳转协议，

>注：如果scheme中存在特殊字符或者中文，则需要先将scheme进行URL Encode，再拼接到跳转协议中。

>>例如，第三方APP的跳转协议为yzjtest://yzjtest?name=jack，则配置跳转地址为：cloudhub://nativeapp?scheme=yzjtest%3a%2f%2fyzjtest%3fname%3djack。其中的scheme值为URLEncode(yzjtest://yzjtest?name=jack)。

><u>注：URL Encode可以自行到第三方网站进行转换，如 http://www.jsons.cn/urlencode</u>

同时，为适配单点登录、下载检测等业务场景，支持扩展以下参数：

>注：以下参数值中如果存在特殊字符或者中文，则需要先将参数值进行URL Encode再拼接到跳转协议中。

>>示例：cloudhub://nativeapp?scheme=yzjtest%3a%2f%2fyzjtest%3fname%3djack&isNeedAuth=false&androidDownloadUrl=https%3a%2f%2fyunzhijia.com%2fhome%2fcustom%2fsaas%2ftest.html%3fcustId%3dlink&packageName=com.yunzhijia.schemehost

2.单点登录

如果跳转协议中配置了isNeedAuth = true 且 appId 配置了具体参数值，则在跳转第三方APP时，

云之家APP会从后端获取ticket信息，拼接到 scheme/iosScheme/androidScheme后面，并跳转至第三方APP。

>比如在上面的例子中，最终跳转到第三方的协议为：yzjtest://yzjtest?name=jack&yzj_ticket=xxxxxxxxxx。

#### 什么是轻应用

轻应用是运行在云之家客户端或Web门户中的`网页程序`。

`平台级轻应用`是云之家提供的通用应用，包括智能审批、智能考勤、时间助手、智能会议等。它们可以开箱即用，也可以与企业已有业务系统进行对接。

`企业自建应用`是企业针对自身业务创建的，它与平台级应用一样，也可以在云之家客户端中运行并利用云之家客户端能力和用户身份。

#### 轻应用如何运行

![轻应用开发概述](/opendocs/file/image/45e26c7df023202a341a303e686a55f0)

名词解释：

```
云之家WebView： 是一个标准浏览器组件，为网页代码提供运行环境。

云之家客户端能力： 包括客户端方法、界面和事件。这些能力只能通过JSSDK（qing.js）来访问。

云之家基础接口： 包括鉴权、获取人员组织等基础接口，以及智能审批等业务接口。
```

# JSSDK

`qing.js` 是轻应用或卡片和客户端之间的桥梁。

通过它，轻应用或者卡片可以调用客户端的函数、唤起客户端的界面，或者监听客户端的状态变化。

#### 引入JS桥

引入在线版本的qing.js，它将会保持最新

```html
<script src="https://static.yunzhijia.com/public/js/qing/latest/qing.js"></script>
```
注意：你不能直接把上面的js保存到本地使用，因为它只是一个下载器，它会根据页面所在的端下载不同的实现js。

#### 调用客户端API

<a href="docs.html#/client-document/index" target="_blank">点击此处</a>查看全部客户端方法

通过 qing.call(方法名, 参数) 可以调用客户端的接口，例如：

```js
// 获取个人信息
qing.call('getPersonInfo', {
    success : function(res){
        alert(JSON.stringify(res))
    }
});
```

上面例子中的 getPersonInfo 是客户端提供的方法之一，它可以获取登录当前客户端的用户的身份信息。

每个客户端接口除了自身必须的参数外，还支持以下**通用参数**：

`success`：接口调用成功时执行的回调函数。

`error`：接口调用失败时执行的回调函数。

`complete`：接口调用完成时执行的回调函数，无论成功或失败都会执行。


以上所述的回调函数的参数说明，如 success : function (res) { ... }，这里的 res 为对象，除了每个接口自身返回的数据之外，还有两个通用属性 code 和 errMsg，其值格式如下：

成功时返回结果示例：`{code: 200, errMsg: "", data: {}}`

失败时返回结果示例：`{code: 500, errMsg: "不合法的图片文件大小", data: {}}`


#### 监听客户端事件

通过 qing.on(事件名, 回调) 可以监听客户端的事件，例如：

```js
qing.on('appear', function () {
    alert('回到当前页面窗口')
})
```

目前支持的事件：

`appear`: 回到当前页面窗口

`disappear`: 离开当前页面窗口

`networkAvailable`: 网络连接变为可用

`networkDisAvailable`: 网络连接变为不可用

`titleDoubleClick`: 标题栏双击

`keyboardChange`: 键盘状态发生变化
