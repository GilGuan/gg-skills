自定义右上角弹出菜单

API

createPop

调用

//0.9.6及以上支持
    qing.call('createPop', {
        'popTitle': String,
        'popTitleCallBackId': String,
        'collapseToItems':false,
        'items': [
            {
                'text': '自定义条目1',
                'callBackId': 'callback1'
            },
            {
                'text': '自定义条目2',
                'callBackId': 'callback2'
            }
        ],
        'menuList': ['forward', 'refresh', 'share', 'openWithBrowser'],
        'shareData': {
            'isShowExt': '转发时是否显示商务伙伴，true or false，默认为true',
            'title': '分享或者转发的标题',
            'url': '分享的链接，若空则取当前的url',
            'description': '分享或者转发的内容',
            'appLogo': '轻应用Logo，base64数据，大小不要超过32k，分辨率推荐198*198，图片格式推荐使用jpg',
            'appName': '轻应用名称'
        },
        'success': function (resp) {
            if (resp.success == true || resp.success == 'true') {
                var callBackId = resp.data ? resp.data.callBackId : '';
                if (callBackId == 'callback1') {
                    callback1();
                }
            }
        }
    });

    function callback1() {
        alert('callback1')
    }

参数说明

回调说明

无

注意:

自定义菜单条目items数量和系统菜单条目menuList数量加起来不超过7条，如超过则优先显示系统菜单条目（比如items数量为5，menuList数量为4，则显示items前3条加menuList里的4条；

菜单显示顺序为优先显示自定义菜单条目，然后显示系统菜单条目，系统条目按JS方法传入的menuList顺序来显示。

移动端10.7.3版本以上，右上角会固定显示“更多”图标，方便用户操作放进悬浮窗功能

## 开发前准备

开发前准备

什么是JS桥

qing.js 是轻应用或卡片和客户端之间的桥梁。

通过它，轻应用或者卡片可以调用客户端的函数、唤起客户端的界面，或者监听客户端的状态变化。

点击此处查看视频教程

引入JS桥

qing.js 是一个加载器，它会自动判断当前网页运行在什么客户端（桌面端、网页端、移动端），然后自动加载适用于当前客户端的、最新的实现代码。

它可以让你的应用能更及时地获得qing.js的新特性和漏洞修复。

<script src="https://static.yunzhijia.com/public/js/qing/latest/qing.js"></script>

> 不要把在线版的 qing.js 下载到本地，否则它将无法正确运行。

> qing.js 并不实现客户端 api，它只负责为 js 访问各端的 api 提供统一接口，因此 qing.js 很少更新。

调用客户端API

通过 qing.call(方法名, 参数) 可以调用客户端的接口，例如：

// 获取个人信息
qing.call('getPersonInfo', {
    success : function(res){
        alert(JSON.stringify(res))
    }
});

上面例子中的 getPersonInfo 是客户端提供的方法之一，它可以获取登录当前客户端的用户的身份信息。

每个客户端接口除了自身必须的参数外，还支持以下通用参数：

success：接口调用成功时执行的回调函数。

error：接口调用失败时执行的回调函数。

complete：接口调用完成时执行的回调函数，无论成功或失败都会执行。

以上所述的回调函数的参数说明，如 success : function (res) { ... }，这里的 res 为对象，除了每个接口自身返回的数据之外，还有两个通用属性 code 和 errMsg，其值格式如下：

成功时返回结果示例：{code: 200, errMsg: "", data: {}}

失败时返回结果示例：{code: 500, errMsg: "不合法的图片文件大小", data: {}}

> 提示：请注意，进入error方法仅代表api没有调通，而如果api调通了但执行结果是false触发的则是success方法；

因此，建议在error和success中都需要进行异常判断，以调用 getLocation 为例：

// 获取定位失败时
function getLocationFailure () {
    // TODO
}

// 获取定位信息
qing.call('getLocation', {
    success: function (result) {
        if (!result || result.success != 'true') {
            getLocationFailure();
            return;
        };

        // 正常逻辑
    },
    error: function (e) {
        getLocationFailure();
    }
});

在表单发起、审批、流转过程中添加自定义行为：

如：在请假单据的发起节点，判断如果用户这个月已经请假三次了，不允许再发起；

cf.ready(function () {

        cf.call('alert', payload)
            .then(function () {
                // 用户点击确定
                var payload = { status: 'success', content: '你点击了确定' }
                var indicator = cf.call('toast', payload)
            })
            .catch(function () {
                // 用户点击取消
                var payload = { status: 'warn', content: '你点击了取消' }
                var indicator = cf.call('toast', payload)
            })
})

# 开发前准备

## 什么是JS桥

`qing.js` 是轻应用或卡片和客户端之间的桥梁。

通过它，轻应用或者卡片可以调用客户端的函数、唤起客户端的界面，或者监听客户端的状态变化。

点击此处查看[视频教程](https://vip.kingdee.com/school/208163815316817152)

## 引入JS桥

 qing.js 是一个加载器，它会自动判断当前网页运行在什么客户端（桌面端、网页端、移动端），然后自动加载适用于当前客户端的、最新的实现代码。

它可以让你的应用能更及时地获得qing.js的新特性和漏洞修复。

```html
<script src="https://static.yunzhijia.com/public/js/qing/latest/qing.js"></script>
```

> 不要把在线版的 qing.js 下载到本地，否则它将无法正确运行。

> qing.js 并不实现客户端 api，它只负责为 js 访问各端的 api 提供统一接口，因此 qing.js 很少更新。

## 调用客户端API

通过 qing.call(方法名, 参数) 可以调用客户端的接口，例如：

```js
// 获取个人信息
qing.call('getPersonInfo', {
    success : function(res){
        alert(JSON.stringify(res))
    }
});
```

上面例子中的 getPersonInfo 是客户端提供的方法之一，它可以获取登录当前客户端的用户的身份信息。

每个客户端接口除了自身必须的参数外，还支持以下**通用参数**：

`success`：接口调用成功时执行的回调函数。

`error`：接口调用失败时执行的回调函数。

`complete`：接口调用完成时执行的回调函数，无论成功或失败都会执行。


以上所述的回调函数的参数说明，如 success : function (res) { ... }，这里的 res 为对象，除了每个接口自身返回的数据之外，还有两个通用属性 code 和 errMsg，其值格式如下：

成功时返回结果示例：`{code: 200, errMsg: "", data: {}}`

失败时返回结果示例：`{code: 500, errMsg: "不合法的图片文件大小", data: {}}`

> 提示：请注意，进入error方法仅代表api没有调通，而如果api调通了但执行结果是false触发的则是success方法；

因此，建议在error和success中都需要进行异常判断，以调用 `getLocation` 为例：

```
// 获取定位失败时
function getLocationFailure () {
    // TODO
}

// 获取定位信息
qing.call('getLocation', {
    success: function (result) {
        if (!result || result.success != 'true') {
            getLocationFailure();
            return;
        };

        // 正常逻辑
    },
    error: function (e) {
        getLocationFailure();
    }
});
```


## 监听客户端事件

通过 qing.on(事件名, 回调) 可以监听客户端的事件，例如：

```js
qing.on('appear', function () {
    alert('回到当前页面窗口')
})
```

目前支持的事件：

`appear`: 回到当前页面窗口

`disappear`: 离开当前页面窗口

`networkAvailable`: 网络连接变为可用

`networkDisAvailable`: 网络连接变为不可用

`titleDoubleClick`: 标题栏双击

`keyboardChange`: 键盘状态发生变化
