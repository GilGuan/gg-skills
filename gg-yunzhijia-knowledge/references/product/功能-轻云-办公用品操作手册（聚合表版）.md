3.业务逻辑

办公用品的核心业务逻辑为物品的库存增减控制，其中涉及库存控制的表有物品入库、物品出库、物品退库、库存调整、物品领用申请、物品退还申请。为了实现对库存的计算和控制，我们建立了一系列聚合表。

使用聚合表计算库存的好处：

3.1.可用库存

可用库存用于计算物品的当前可用库存，当前可用库存需要考虑在途申领和退还的数量：

可用库存 = 入库数量 - 出库数量 + 退库数量 - 领用数量 + 退还数量 + 调整数量

当前可用库存决定了可以扣减的物品数量。

可用库存需要考虑在途的申领数量和退还数量，但流程状态为已废弃、不同意的物品领用申请和物品退还申请数据不需要考虑在内，所以针对可用库存的聚合表设置了数据过滤条件：

物品领用申请：流程状态 包含 已完成、已挂起、审批中、待提交

物品退还申请：流程状态 包含 已完成、已挂起、审批中、待提交

例如签字笔当前库存中有500支，但有10支处于在途申请流程中，所以签字笔的可用库存为490支（500-10）。

在可用库存聚合表上设置一个提交校验规则，用于确保物品领用或出库数量不会超过可用数量。

可用库存不足！：可用库存 < 0

3.2.即时库存

即时库存可以反应当前真实库存，即时库存的计算逻辑与可用库存一致。即时库存不能用来控制可以扣减的物品数量。

一般来讲，物品的可用库存会小于等于即时库存。

对于物品领用申请和物品退还申请，即时库存仅考虑流程状态为已完成的数据，其他状态的数据统统不计入即时库存。

例如签字笔当前库存中有500支，有10支处于在途申请流程中，但签字笔的即时库存仍为500支（因为这10支尚处于申请过程中，不计入最终库存）

在即时库存聚合表上设置了一个提交校验规则，用于确保物品领用或出库数量不会超出可用数量。

当前库存不足：当前库存 < 0
